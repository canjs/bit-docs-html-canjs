      Welcome to the CanJS docs! Learn about all the packages that make-up CanJS & how they work together to help you build amazing applications!

      Collections

      The can Package

      While CanJS is designed to be used and should be used as independent packages, we still publish a can package with a version number, currently 3.8.1. We do this so there is a specified version of all of the sub-project packages that are integration tested to work together.

      The can package uses semantic versioning with respect to the Core and Infrastructure collections only. Read the Release Guide for more information on CanJS’s release process.

      The sub-project package versions for can 3.8.1 are listed below. Copy the ones you use into your package.json if you want to use the same packages that were tested as part of CanJS 3.8.1’s release.
